# Binance Local Watcher TODO

更新日時：2026-06-01 JST

## 現在の位置づけ

Electron + Node local engine版を中心に進行中。  
目的は、利益保証や売買指示ではなく、手動取引前に **無理な取引を避けるための条件診断・準備・記録** を支援すること。

現在の主工程は、**アラートまわりの土台整理とUI改善**。  
チャート・アラート・Daily Goal・約定可能性を混ぜず、次の役割分担で整理する。

- チャート：見る画面
- アラート：気づく画面
- Daily Goal：条件診断画面
- 約定可能性：別計算
- データ保存：表示目的ではなく、分析・診断用データ準備

重要方針：

- 実注文はまだ入れない
- 自動売買はしない
- 出金機能は入れない
- APIキーやSecretは保存しない
- アプリは売買指示をしない
- AIやアラートは売買判断を代行しない
- `.csv` は最低限利用に留め、分析・診断の本体はSQLite/分析キャッシュへ寄せる
- `data/blw.sqlite` などローカル実行データはGitHubに上げない

DL保存は、単にチャートを表示するためではなく、分析・診断に使うデータを準備するための操作とする。

### 公開kline一時表示

- 表示用
- 保存しない
- 診断根拠には使わない

### DL済み＋現在まで更新済みkline

- 分析用キャッシュ
- 仮想約定率
- 必要値幅出現率
- 日次目標診断
- アラート判定
- 参照根拠表示

### データ保持方針

- BTCJPY / ETHJPY の1分足を中心に保存
- 標準保持は30日
- 標準参照は7日
- 必要に応じて14日 / 30日参照
- 30日を超えるデータは、将来削除または集計保存

### チャート・アラート・約定可能性の分離

- チャート表示時間足：見るための表示設定
- アラート評価窓：1m / 5m / 15m / 30m / 1h の価格変化チェック
- 約定可能性：1分足キャッシュを使った別計算
- Daily Goal：条件診断と現実度の確認

### 日次目標タブで分けるもの

- 仮想約定率：指値が価格到達した想定
- 必要値幅出現率：必要値幅が過去データ上で出た頻度
- 必要勝率：目標達成に必要な勝ち方の厳しさ
- 到達想定：勝ち回数ではなく、指値に到達した想定回数
- 1回あたり必要値幅：1回の取引で利益を残すために必要な値幅
- 指値候補診断：入口価格、必要利確価格、到達可能性、損切りリスクの比較

### アラートの位置づけ

- アラートは売買サインではない
- アラートは手動取引前に「気づく」ための補助
- 小さすぎる値動きは情報表示寄り
- 注意以上は、手数料・スプレッド・滑りより意味のある値動きを基準にする
- 将来的には過去データからおすすめしきい値を算出する

### 出来高の位置づけ

- 最初は表示のみ、またはアラート補助
- 出来高だけで売買判断しない
- 絶対出来高より、直近平均・中央値との相対出来高を重視する

## 次に進むなら

1. アラートタブの実機表示を確認する
2. メルヘン配色をもう少し整える
3. アラート判定サマリーを上部に追加するか判断する
4. アラート履歴保存の条件を確認する
5. その後、指値候補診断と必要値幅ロジック再設計へ進む
